## Spacing Scale

Declared values (must be multiples of 4). Pre-populated from existing codebase patterns in `ManualConnectionDialog.axaml` and `ConnectionIndicator.axaml`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Connection indicator margin (`Margin="0,4"`), icon gaps |
| sm | 8px | Button row spacing (`Spacing="8"`), inline element gaps, tag padding |
| md | 16px | Stack panel item spacing (`Spacing="16"`), TextBox/input padding |
| lg | 24px | Dialog/card padding (`Padding="24"`), section body padding |
| xl | 32px | Section-to-section gap in scroll body |
| 2xl | 48px | Major layout breaks (not yet used; reserved) |
| 3xl | 64px | Page-level vertical centering (not yet used; reserved) |

Exceptions:
- Section header bottom border: 1px, not a spacing token
- Touch targets (radio buttons, toggle switches): minimum 44px height on Android
- Save bar sticky row: 64px height (`RowDefinition Height="64"`) to accommodate button + vertical padding
- CornerRadius: 8px for cards and dialogs (from `ManualConnectionDialog.axaml`); 4px for inline badges (from `ConnectionIndicator.axaml`)

## Typography

Pre-populated from observed font sizes in `ManualConnectionDialog.axaml` and `ConnectionIndicator.axaml`. Source: CONTEXT.md D-04, codebase scan.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Caption / status text | 12px | Regular (400) | 1.4 |
| Body / label | 14px | Regular (400) | 1.5 |
| Input / interactive | 16px | Regular (400) | 1.5 |
| Section heading | 18px | SemiBold (600) | 1.2 |

**Notes:**
- Only two weights are used: Regular (400) and SemiBold (600). SemiBold is reserved for section headings and dialog titles only.
- The existing codebase uses `FontSize="13"` for status text and `FontSize="18" FontWeight="SemiBold"` for dialog titles. Body/label content uses `FontSize="13"` or `FontSize="16"`. This contract standardises body at 14px (slightly increased from 13px for settings form readability) and retains 16px for interactive controls.
- Error message text (form validation): 12px Regular.
- Section headers (`TextBlock` acting as category divider): 18px SemiBold — same as dialog title pattern.

## Color

Pre-populated from FluentTheme Dark + observed hex values in codebase. Source: `App.axaml` (FluentTheme Dark), `ManualConnectionDialog.axaml`, `ConnectionIndicator.axaml`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | FluentTheme Dark background token (`#1C1C1C` effective) | Page background, ScrollViewer body |
| Secondary (30%) | `#1EFFFFFF` (5% white overlay on dark) / FluentTheme card surface | Section header rows, Save bar background, card/border backgrounds |
| Accent (10%) | FluentTheme accent (Blue `#0078D4` effective — Fluent default) | Save button (Classes="accent"), dirty-state Save bar highlight |
| Destructive | `#FFF44336` (Material Red 400 — established in codebase) | Danger Zone section border, Factory Reset button, SD Card Wipe button, error feedback text |

Accent reserved for:
- Save button (the primary CTA — only appears when dirty state is true)
- Active tab indicator (inherited from FluentTheme TabbedPage)

Destructive colour reserved for:
- Danger Zone section container border/background tint
- Factory Reset action button
- SD Card Wipe action button
- Form validation error text (already established pattern)
- Save failure toast/feedback text

**Danger Zone visual separation (D-06):** The Danger Zone section uses a `Border` with `BorderBrush="#FFF44336"` at `BorderThickness="1"` and a subtle background tint `Background="#19F44336"` (10% opacity red) to signal destructiveness. All other sections use no explicit border.

**Layout structure (from D-05, RESEARCH.md):**
```
ContentPage
  Grid RowDefinitions="*, 64"
    Row 0: ScrollViewer (all setting sections)
      StackPanel Spacing="32"
        [Section: WiFi]
        [Section: Recording]
        [Section: Channels]
        [Section: Camera]
        [Section: Sensors]
        [Section: System]
        [Section: Overlays]
        [Section: Danger Zone]  ← red-bordered
    Row 1: Save bar (IsVisible when dirty)
      Border Background="secondary surface"
        StackPanel Orientation="Horizontal" HorizontalAlignment="Right"
          Button Content="Save Settings" Classes="accent"
```

**Section header pattern:** Each section opens with a `TextBlock FontSize="18" FontWeight="SemiBold"` followed by a `Separator` (`Height="1"`) and then `Spacing="16"` item rows.

**Setting row pattern:** Each setting is a `Grid ColumnDefinitions="*, Auto"` where the label is left-aligned body text and the control is right-aligned.

## Interaction States

### Settings Page

| State | What user sees |
|-------|----------------|
| Loading | Spinner / "Loading settings..." centered in scroll area; controls not shown |
| Loaded (clean) | All sections visible; Save bar hidden |
| Loaded (dirty) | All sections visible; Save bar slides up at bottom |
| Saving | Save button shows loading indicator; controls disabled during save |
| Save success | Save bar shows "Settings saved." in place of button; reverts to hidden after 3s |
| Save failure | Save bar shows "Failed to save settings. Try again."; Save button re-enabled |
| Navigate away (dirty) | "Unsaved Settings" dialog appears before navigation proceeds |

### Confirmation Dialogs

| State | What user sees |
|-------|----------------|
| Dialog open | Modal overlay; Cancel (secondary) + confirm (destructive / accent) buttons |
| Confirm tapped | Dialog closes; operation executes; loading state shown if long-running |
| Cancel tapped | Dialog closes; no action taken; user returns to Settings page |
