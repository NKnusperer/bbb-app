## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, chip internal gaps |
| sm | 8px | Between metadata fields within a card row, chip bar gap, player control icon gaps |
| md | 16px | Card internal padding, page horizontal margin, StackPanel section spacing |
| lg | 24px | Between major sections (filter bar to list, metadata blocks in detail view) |
| xl | 32px | Section header to content gap in detail view, StackPanel Spacing on settings-style pages |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Thumbnail size: 160x90px (16:9 ratio, fixed; not a spacing token but a layout constant)
- Touch targets for player controls: minimum 44px hit area (pad with invisible hit area, visual size may be smaller)
- Chip bar height: 36px total (8px vertical padding + 20px text = 36px)
- Progress bar height: 4px (matches established SettingsPage loading indicator style)
- Nested clip indent inside trip group: 24px left margin from card left edge

Source: Established from `SettingsPage.axaml` (`Margin="16"`, `Spacing="32"`, `Spacing="16"`, `Spacing="8"`) and CONTEXT.md D-04 (thumbnail 160x90).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (Regular) | 1.5 |
| Label | 13px | 400 (Regular) | 1.4 |
| Heading | 18px | 600 (SemiBold) | 1.2 |
| Display | 24px | 600 (SemiBold) | 1.2 |

Usage mapping:
- **Body (14px/400):** Card metadata text (date/time, duration, file size, speed, G-force, distance), detail view metadata values, player time display, archive progress label, section body text
- **Label (13px/400):** Connection indicator text (established in `ConnectionIndicator.axaml` FontSize="13"), chip filter labels, metadata field keys, badge text, secondary captions
- **Heading (18px/600):** Section headers in detail view (e.g., "Front Camera", "Rear Camera", "Recording Details"), trip group header title, page section dividers — matches established SettingsPage `FontSize="18" FontWeight="SemiBold"`
- **Display (24px/600):** Placeholder page title (existing RecordingsPage uses FontSize="24") — retained for NavigationPage pushed header or empty state heading

Source: `SettingsPage.axaml` (FontSize 14 for body fields, 18/SemiBold for section headers); `ConnectionIndicator.axaml` (FontSize 13 for label); `RecordingsPage.axaml` (FontSize 24 placeholder).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | System FluentDark surface (~#1E1E1E) | Page backgrounds, ScrollViewer area, list background — inherits from FluentTheme Dark |
| Secondary (30%) | #1EFFFFFF (12% white overlay on dark) | Recording cards, trip group header cards, player control bar background, metadata panels, chip bar background — matches established `Border Background="#1EFFFFFF"` pattern |
| Accent (10%) | #2196F3 (Material Blue 500) | Active filter chip selection indicator, seek bar fill, playback speed active selection, archive progress bar fill, multi-select checkboxes |
| Destructive | #F44336 (Material Red) | No destructive actions in this phase (archive is additive, not destructive) — defined for consistency with established `Button.destructive` style; not used in primary flows |

Accent reserved for:
1. Active/selected filter chip (the one currently selected in All | Radar | G-Shock | Parking bar)
2. Seek bar fill (the played portion of the seek track)
3. Archive progress bar fill
4. Multi-select checkboxes (checked state)
5. Playback speed active selection highlight

Semantic colors (not accent):
- **Event type badges — each badge has its own semantic color, not accent:**
  - None / normal: #666666 (neutral gray)
  - Radar: #FFC107 (amber — matches Searching state from `ConnectionStateToBrushConverter`)
  - G-Shock: #F44336 (red — matches Disconnected state; high-severity event)
  - Parking: #FF9800 (orange — matches NeedsProvisioning state)
  - Archived badge: #4CAF50 (green — matches Connected state; "safe" indicator)
- **Status semantic colors (from existing `ConnectionStateToBrushConverter`):**
  - Searching: #FFC107
  - Connected: #4CAF50
  - NeedsProvisioning: #FF9800
  - Disconnected: #F44336

Source: `ConnectionStateToBrushConverter.cs` (semantic color palette), `App.axaml` (`Button.destructive` = #F44336), `ConfirmDialog.axaml` (`Background="#FF1E1E1E"`), `ConnectionIndicator.axaml` (`Background="#1EFFFFFF"`).

## Component Inventory

Components needed for this phase, expressed as Avalonia controls and custom templates:

### Recording List Page (`RecordingsPage.axaml`)

**Filter Chip Bar**
- Container: `StackPanel Orientation="Horizontal" Spacing="8"` inside `ScrollViewer HorizontalScrollBarVisibility="Auto"` (allows overflow on very small screens)
- Chip: `Button` with `CornerRadius="16"` and `Padding="12,8"` (8px vertical, 12px horizontal)
- Inactive chip: `Background="#1EFFFFFF"` border-style button, `Foreground` from theme
- Active chip: `Background="#2196F3"` (accent), `Foreground="White"`
- Font: 13px/400
- Behavior: single-select; bound to `SelectedFilter` enum property on RecordingsViewModel

**Recording Card** (standalone clip)
- Container: `Border CornerRadius="8" Padding="12" Background="#1EFFFFFF"` — 8px corner radius, 12px padding (md spacing)
- Layout: `Grid ColumnDefinitions="160,*"` — thumbnail left, metadata right
- Thumbnail: `Image Width="160" Height="90" Stretch="UniformToFill"` — bound to `ThumbnailData` as `BitmapImage`; placeholder = solid color block per D-04
- Metadata right column: `StackPanel Spacing="4"` — two rows of info
  - Row 1: date/time (14px/400) + duration + file size, `Opacity="0.85"`
  - Row 2: speed + G-force + distance, `Opacity="0.7"`
- Event badge: `Border CornerRadius="4" Padding="4,2"` with `Background` from `EventTypeToBrushConverter`, `TextBlock` 13px/400, positioned top-right corner of thumbnail via `Panel`
- Archived badge: `Border CornerRadius="4" Padding="4,2" Background="#4CAF50"` with checkmark icon + "Archived" label, bottom-right of card
- Multi-select checkbox: `CheckBox` overlaid top-left of thumbnail via `Panel`, visible only in multi-select mode

**Trip Group Header Card**
- Container: `Border CornerRadius="8" Padding="12" Background="#26FFFFFF"` (slightly more prominent than clip cards — 15% white vs 12%)
- Trip icon + "Trip" label: `StackPanel Orientation="Horizontal" Spacing="8"` with PathGeometry route icon
- Aggregated stats: same two-row metadata layout as recording card
- Clip count: "N clips" label, 13px/400, Opacity="0.7"
- Expand/collapse chevron: right-aligned `PathGeometry` icon, rotates 90deg when expanded

**Nested Clip Row** (inside trip group)
- Container: `Border Margin="24,0,0,0"` (24px left indent from trip group) + same card styling at reduced opacity or border-only style
- Same layout as standalone recording card but without trip badge
- Smaller thumbnail: 120x68px (maintains 16:9 ratio, reduces visual weight in nested context)

**Loading State**
- `ProgressBar IsIndeterminate="True" Width="160" Height="4"` centered — matches SettingsPage pattern
- `TextBlock Text="Loading recordings..." FontSize="14"` below, `HorizontalAlignment="Center"`

**Empty State** (see Copywriting section for text)
- `StackPanel` centered, `Spacing="16"`
- PathGeometry filmstrip/recording icon, `Width="64" Height="64" Opacity="0.4"`
- Heading: 24px/600
- Body: 14px/400, `Opacity="0.7"`, `MaxWidth="280"` centered

**Error State** (see Copywriting section for text)
- Same layout as SettingsPage error state: heading 18px/600 + body 14px/400 centered

### Recording Detail Page (`RecordingDetailPage.axaml`)

**Back Navigation**
- NavigationPage provides built-in back button via `NavigationPage.PushAsync` (D-10)
- No custom back button needed

**Video Player Area**
- Desktop (landscape ≥ 600px): `Grid ColumnDefinitions="*,*" ColumnSpacing="8"` — two `VideoView` controls side-by-side
- Mobile (portrait < 600px): `StackPanel Orientation="Vertical" Spacing="8"` — two `VideoView` controls stacked
- Single-camera recording: full-width single `VideoView`, no grid
- `VideoView` aspect ratio: 16:9, bound to `NativeControlHost`
- Adaptive layout: `AdaptiveBehavior` + `AdaptiveClassSetter` on root panel, breakpoint 600px (established in Phase 1)

**Player Controls**
- Container: `StackPanel Spacing="8"` below VideoView area, full width
- Row 1 (seek): `Slider` with `Background="#2196F3"` track fill, `Padding="0,4"`, full width
  - Time labels: current/total displayed as `TextBlock 13px/400` flanking the slider
- Row 2 (buttons): `StackPanel Orientation="Horizontal" Spacing="8" HorizontalAlignment="Center"`
  - Previous Frame: PathGeometry icon button, 44px hit area
  - Play/Pause: PathGeometry icon button, 44px hit area (larger visual — 32px icon)
  - Next Frame: PathGeometry icon button, 44px hit area
  - Fullscreen Toggle: PathGeometry icon button, 44px hit area, right-aligned
- Row 3 (speed): `StackPanel Orientation="Horizontal" Spacing="4" HorizontalAlignment="Center"`
  - Speed buttons: `0.5x` `1x` `2x` as toggle-style `Button` controls, 13px/400
  - Active speed: `Background="#2196F3"` (accent)

**Metadata Section**
- `StackPanel Spacing="24" Margin="16"` (lg spacing between blocks)
- Section heading: 18px/600 ("Recording Details" or "Trip Details")
- Metadata grid: `Grid ColumnDefinitions="*,*"` for label + value pairs, 14px/400 for values
- For trips: additional "Clips in this trip" section with compact clip list

**Archive Controls**
- Single recording: `Button Content="Archive Recording"` (primary style), full width, `Margin="16"`
- Trip detail: `Button Content="Archive Entire Trip"` + `Button Content="Select Clips to Archive"` stacked with `Spacing="8"`
- Multi-select mode: `Button Content="Archive Selected (N)"` (accent background), disabled when nothing selected

**Archive Progress Overlay**
- Semi-transparent overlay: `Border Background="#CC000000"` full-screen
- Inner: `StackPanel` centered with `Spacing="16"`
  - Label: 14px/400 "Archiving clip N of M..."
  - `ProgressBar Value="{Binding ArchiveProgress}" Height="4"` with accent fill
  - `Button Content="Cancel"` (text style, no fill)
