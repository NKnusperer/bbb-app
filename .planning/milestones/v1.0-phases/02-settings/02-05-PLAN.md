    Per D-13 and Research Pitfall 6: Implement tab navigation guard in AppShellViewModel by monitoring tab index changes. When the user navigates away from the Settings tab while SettingsVm.HasUnsavedChanges is true, show the unsaved-changes dialog.

    **Add to AppShellViewModel:**

    1. Add `SelectedTabIndex` observable property for two-way binding with TabbedPage:
    ```csharp
    [ObservableProperty]
    private int _selectedTabIndex;
    ```

    2. Track which tab index is Settings. Based on AppShellView.axaml tab order: Dashboard=0, Recordings=1, LiveFeed=2, Settings=3.
    ```csharp
    private const int SettingsTabIndex = 3;
    private int _previousTabIndex;
    private bool _isHandlingNavigation;
    ```

    3. Use `partial void OnSelectedTabIndexChanged(int oldValue, int newValue)` (CommunityToolkit generates this for `[ObservableProperty]`):
    ```csharp
    partial void OnSelectedTabIndexChanged(int oldValue, int newValue)
    {
        if (_isHandlingNavigation) return;

        // Only guard when leaving Settings tab
        if (oldValue == SettingsTabIndex && SettingsVm.HasUnsavedChanges)
        {
            _previousTabIndex = oldValue;
            _ = HandleUnsavedChangesNavigationAsync(newValue);
        }
    }

    private async Task HandleUnsavedChangesNavigationAsync(int requestedIndex)
    {
        _isHandlingNavigation = true;

        // Revert to Settings tab immediately (prevent navigation)
        SelectedTabIndex = SettingsTabIndex;

        var shouldSave = await DialogService.ShowConfirmAsync(
            "Unsaved Settings",
            "You have unsaved settings. What would you like to do?",
            confirmText: "Save & Leave",
            cancelText: "Discard Changes",
            isDestructive: false);

        if (shouldSave)
        {
            if (SettingsVm.SaveCommand.CanExecute(null))
                await SettingsVm.SaveCommand.ExecuteAsync(null);
        }
        else
        {
            SettingsVm.DiscardChanges();
        }

        SelectedTabIndex = requestedIndex;
        _isHandlingNavigation = false;
    }
    ```

    **Add DiscardChanges to SettingsViewModel** — public method to clear dirty state without saving:
    ```csharp
    public void DiscardChanges()
    {
        _isDirty = false;
        OnPropertyChanged(nameof(HasUnsavedChanges));
        SaveCommand.NotifyCanExecuteChanged();
    }
    ```

    **Update AppShellView.axaml** — bind TabbedPage SelectedIndex:
    Add `SelectedIndex="{Binding SelectedTabIndex, Mode=TwoWay}"` to the `<TabbedPage>` element:
    ```xml
    <TabbedPage Grid.Row="1" TabPlacement="Auto"
                SelectedIndex="{Binding SelectedTabIndex, Mode=TwoWay}">
    ```

    **Write tests** (add to AppShellViewModelTests.cs):
    - `OnSelectedTabIndexChanged_FromSettingsWithUnsavedChanges_ShowsDialog` — change index from 3 to 0, verify ShowConfirmAsync called
    - `OnSelectedTabIndexChanged_FromSettingsWithNoChanges_DoesNotShowDialog` — change index from 3 to 0, verify ShowConfirmAsync NOT called
    - `OnSelectedTabIndexChanged_FromNonSettingsTab_DoesNotShowDialog` — change index from 0 to 1, verify no dialog
    - `HandleUnsavedChanges_UserChoosesSave_CallsSaveCommand` — dialog returns true, verify SaveCommand executed
    - `HandleUnsavedChanges_UserChoosesDiscard_CallsDiscardChanges` — dialog returns false, verify tab navigates without save

    Use a real SettingsViewModel with NSubstitute mocks for IDashcamDevice and IDialogService for realistic behavior in tests.
